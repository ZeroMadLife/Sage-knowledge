# V6.9 阶段产出与 Git 收口记录

> 本章只记录已经进入 Git 历史或已通过验证的实现。设计书中的目标会单独标为“未交付”，不能混写。

## 这阶段真正做成了什么

### 1. 上下文与证据底座

- 结构化 transcript、上下文 projection 与受限 compaction 已进入 `dev/sage-v6`；压缩不会覆盖 canonical transcript。
- SQLite evidence/transcript、迁移、恢复和导出链路已落地；记忆 proposal 有 revision、幂等 projection 与审核 UI/API 基础。
- 这些能力让“当前上下文”和“长期可审计的事实”不再依赖单一内存对象。

### 2. V6.9 持久时间线与运行重连

集成分支 `codex/sage-v6-9-integration` 已实现：

- `SessionEventJournal`：浏览器可见事件持久化，带单调 `sequence` 与稳定 `event_id`；
- `RunCoordinator`：run 生命周期由服务端持有，WebSocket 不再是运行本体；
- timeline REST 补页与 WebSocket `after` 补发，前端按 session 分区并按事件 ID 去重；
- 会话元数据与历史过程投影，为刷新、回到旧会话和晚到事件提供状态基础。

本质上的变化是：实时推送从唯一数据来源变成通知，持久事件和 REST 回放才是事实来源。

### 3. 中文 Coding Workbench

UI worktree `codex/v6-9-hermes-workbench` 在 Hermes Studio 的信息架构启发下独立实现，未复制对方源码、样式、资源或品牌内容：

- 左侧会话/工作区导航，支持新建、选择、搜索、重命名、置顶、归档与恢复；
- 中间统一 `CodingMessageTurn` 时间线，工具过程、审批、记忆提案与回答可在同一 turn 下展开；
- 右侧可收起、可调宽 Inspector，承载 Files、Changes、Runs、Memory；
- 以 `/coding/session/:id` 深链打开会话，保留滚动位置，移动端使用全屏 sheet。

## 当前 Git 事实

| 位置 | 分支 | 状态 | 结论 |
| --- | --- | --- | --- |
| 主工作区 | `dev/sage-v6` | 当前集成目标，有待纳入的中文设计/计划文档 | 保留，不清理用户材料 |
| 集成 worktree | `codex/sage-v6-9-integration` | 时间线/运行重连后端与前端基础已提交 | 暂不关闭，等待 UI 最后修复合入并做整体验证 |
| UI worktree | `codex/v6-9-hermes-workbench` | 工作台功能已提交；运行重连和键盘可访问性修复正在收尾 | 暂不合并或删除 |

分支之间不能仅凭“功能看起来相似”就删除。必须先确认目标分支已经包含短期分支的最终提交，且短期 worktree 没有未提交变更。

## 本轮剩余的关闭门禁

UI worktree 的最后修复完成后，依次执行：

1. UI worktree：完整前端测试、生产构建、`git diff --check`；
2. 合并到 V6.9 集成 worktree：解决真实冲突后重跑前后端受影响测试；
3. 浏览器验收：`1440x900`、`1024x768`、`390x844` 下检查深链、切会话、断线重连、工具过程回放、长路径和滚动恢复；
4. 复审：确认运行中切换会话不停止服务端 run，恢复后终态只出现一次；
5. 只有通过以上门禁，才回合 `dev/sage-v6`；短期 worktree 此时可以删除。

V6.9 仍不能进入 `main`，因为 Skills 权限路由、只读 Explore/Reviewer、`/remember` SQLite canonical facts 和 Dream 受限反思闭环尚未完全交付。

## 每次 Git 后的固定复盘模板

```markdown
## Git 收口复盘

- source commit：
- 需求完成度：已完成 / 未完成边界
- 验证：测试、构建、静态检查、浏览器场景
- 集成判定：可合并 / 继续开发 / 需修复
- worktree 判定：保留 / 合并后删除
- 已关闭风险：
- 遗留风险：
- 下一阶段：
```

这不是行政记录。它用来防止三种常见错误：把未完成分支误删、把未验证分支直接推到发布分支、以及过一周后不知道每个 worktree 为什么还存在。
