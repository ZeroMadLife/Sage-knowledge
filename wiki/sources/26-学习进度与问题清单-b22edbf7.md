## Git 与 Worktree 收口规则

- `main` 只保留可发布版本；当前集成目标为 `dev/sage-v6`；功能/修复使用短期 `codex/*` worktree 分支。
- 每次 git 提交或功能完成后，必须回顾：需求是否完成、测试/构建是否通过、是否能安全合入集成分支、共享边界是否经过审查。
- 合并进集成分支并重新验证后，确认短期分支是祖先且 worktree 干净，立即删除该短期 worktree 与对应本地分支。
- 不删除主工作区中用户未跟踪的设计文档；清理前先区分用户材料与已合入的短期开发产物。
- 每次合并都写入本 Obsidian 学习库：source commit、验证证据、关闭的风险和下一阶段边界。
- 每次短期分支完成后都使用 [[30-V6.9阶段产出与Git收口记录]] 的模板作出可合并、保留或删除的明确判定。
