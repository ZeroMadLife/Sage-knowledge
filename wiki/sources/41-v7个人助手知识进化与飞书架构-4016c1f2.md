# V7 个人助手知识进化与飞书架构

## 阶段结论

V6 在 `4de761b` 冻结，新的长期集成分支为 `dev/sage-v7`。Sage 的主产品不再是单纯 Coding Agent，而是面向程序员和技术学习者的 Personal AI Learning Companion；Coding Runtime 作为 Practice Engine 继续复用。

完整设计见：

- `/Users/zeromadlife/Desktop/tour-agent/docs/superpowers/specs/2026-07-15-sage-v7-personal-assistant-knowledge-evolution-design.md`
- `/Users/zeromadlife/Desktop/tour-agent/docs/superpowers/plans/2026-07-15-sage-v7-assistant-home.md`

## 已确认产品决策

- 首页采用左侧能力菜单、中间 Chat 的对话式个人主页；
- 左侧入口为今天、历史对话、项目、知识库、成长记录、公开主页和设置；
- 首页显示真实项目、知识连接和待确认提案，不用假数据伪装 RAG 已完成；
- 颜色使用 Green/Blue/Yellow/Coral 表达同步、来源、审核和风险；
- 知识采用 LLM Wiki 式持久 Markdown 产物；
- 原始来源、Wiki、检索索引、Durable Memory 和 Learning State 保持独立；
- 所有知识与记忆更新坚持 proposal-first；
- HR Agent 使用独立公开资料包和独立公开索引；
- 飞书开发机器人与 Sage 产品机器人分离；
- 首次上线使用 Docker Compose + GitHub Actions + 单台服务器。

## LLM Wiki 借鉴边界

采用 Karpathy LLM Wiki pattern 的抽象思想：

```text
immutable raw sources
  -> LLM-maintained Markdown wiki
  -> schema/purpose rules
  -> index.md + log.md + [[wikilink]] + frontmatter
```

参考实现 `nashsu/llm_wiki` 使用 GPLv3，当前是 TypeScript 桌面应用与 Rust 后端 Agent 的组合。Sage 只借鉴两阶段摄取、持久队列、来源追溯和 Wiki 维护等公开行为，不复制其源码、UI、资源或内部实现；数据契约和代码独立完成。

## 受控进化闭环

```text
授权来源
  -> Ingest / Retrieval
  -> 带引用回答或实践结果
  -> Query Note / Wiki / Memory Proposal
  -> 用户编辑和批准
  -> 新 revision
  -> 增量重建索引
```

“自我进化”是可追溯、可批准、可回滚的知识更新过程，不是模型静默修改自己。

## 飞书边界

开发机器人：

```text
指定私有群
  -> cc-connect WebSocket
  -> Codex CLI
  -> thread-isolated session
  -> per-task worktree
  -> 测试 / diff / 审批回传
```

只允许指定账号和群；提交、合并、危险命令与部署必须批准；禁止直接修改 `main`。cc-connect 启动的是新的 Codex CLI 会话，不是接管当前 Desktop 任务。连接器安装、飞书应用配置和联调由用户独立完成，不阻塞 Sage V7-P1 至 P3。

2026-07-15 已先以 Claude Code 跑通开发机器人的最小只读链路，完整证据与安全边界见
[[44-飞书cc-connect与ClaudeCode链路收口]]。Codex CLI、受控写入、per-task worktree
与部署审批仍属于后续阶段。

产品机器人只调用 Sage API，用于知识问答、状态查询和飞书卡片审批，不获得 shell、Git 或部署权限。
