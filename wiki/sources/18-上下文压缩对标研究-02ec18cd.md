# 上下文压缩对标研究

> [!abstract] 结论
> Sage 不能继续只做字符级 `tail_clip`。比较稳妥的方向是：工具结果先分层减重，达到门限后只在 turn boundary 做语义压缩，原始 transcript 永久保留，压缩失败时保留旧上下文并熔断。

## Sage 当前是什么

当前 `ContextManager` 使用 60000 字符预算，优先保留 current request、skill 和 memory，再裁剪 history 与 prefix。`CompactManager` 虽然能把旧 turn 折成确定性摘要，但目前只在测试中使用，没有接入 Runtime 的自动触发链。

因此当前行为是：

```text
完整 history 持续增长
  -> 每轮 render prompt 时截取 history 尾部
  -> 没有 token usage 反馈
  -> 没有自动 compact
  -> 没有压缩前记忆 flush
  -> 没有摘要失败恢复与熔断
```

这会丢掉旧任务状态，但又没有生成可继续执行的 handoff。

## Claude Mini 的分层策略

Claude Mini 的教学版和 production 版必须分开看。教学快照是“消息超过 6 条就摘要旧消息、保留最近 2 条”，只适合讲概念；production CLI 才有完整流水线。

| 层级 | 触发条件 | 行为 |
| --- | --- | --- |
| 大结果落盘 | tool result > 30KB | 全文写入磁盘，上下文只保留前 200 行和恢复路径 |
| 单结果兜底 | result > 50000 chars | 保留头尾并标记截断 |
| Budget | utilization >= 50% | 历史 tool result 上限 30000 chars |
| Budget 高压 | utilization > 70% | 历史 tool result 上限降到 15000 chars |
| Snip | utilization >= 60% | 清理重复 read/search，始终保留最近 3 个结果 |
| Cache 保护 | cache 热且 utilization < 75% | 暂不改写缓存前缀 |
| Microcompact | idle >= 5 分钟 | 清空旧 tool result，保留最近 3 个 |
| Auto compact | effective window 的 85% | 摘要旧历史；200K 模型约等于总窗口 76.5% |

`effectiveWindow = modelContextWindow - 20000 output reserve`。自动 compact 只能发生在最新 plain user message 之后、下一次模型调用之前；不能在工具循环中途切断 `tool_use/tool_result` 配对。

值得采用的是分层减重、缓存感知和 turn boundary。不能照搬的部分包括：非原子覆盖 session JSON、unknown model 默认按 200K、压缩失败无熔断，以及部分被截断工具结果没有先落盘。

## Hermes Agent 的强化点

Hermes 默认以有效输入窗口的 50%作为正常压缩起点，扣除输出 token 预留，并为小窗口设置 85%安全退化门限。它比 Claude Mini 更强调可恢复性：

- 先清旧 tool result，再按 token 预算选 recent tail。
- 默认保护开头 3 条非 system 消息和至少 20 条尾部消息；实际尾部以 token 预算为主。
- 摘要上限为 `min(context * 5%, 12K tokens)`。
- 复用上一次摘要做 iterative summary。
- 压缩时不拆工具调用组，并修复角色交替。
- 同一 session 通过租约避免并发压缩。
- 旧 transcript soft archive，压缩后的 active transcript 可继续执行。
- 连续两次压缩节省不足 10%时停止自动重压。

Hermes 默认在部分摘要失败时可能写静态 fallback 并丢弃中段。Sage 应选择更保守的配置：摘要失败就保留原上下文，不允许为了“继续跑”而静默丢历史。

## OpenClaw 的强化点

OpenClaw 的 `safeguard` compaction 重点是分块、分阶段和质量修复：

- 完整历史留在磁盘，模型只看到 summary + recent tail。
- 单块摘要最多重试 3 次，可保留 partial summary。
- 默认原样保留最近 3 个完整 turn，上限 12 个。
- summary 默认上限 16K 字符。
- quality guard 默认修复 1 次，最多 3 次。
- 压缩前先做 silent memory flush，避免长期信息只存在于一次摘要里。

## Sage 应采用与拒绝的部分

| 决策 | 结论 | 原因 |
| --- | --- | --- |
| 从字符预算改为 token/effective-window 预算 | 采用 | 与真实模型限制一致 |
| 工具结果先落盘再减重 | 采用 | 保留可恢复证据 |
| turn boundary 自动压缩 | 采用 | 不破坏工具调用协议 |
| 原始 transcript archive | 采用 | 摘要不是事实源 |
| iterative structured summary | 采用 | 多次压缩仍能连续执行 |
| 摘要失败静态 fallback 后丢中段 | 拒绝 | 可能永久丢任务事实 |
| 每轮都重建动态 system prompt | 拒绝 | 破坏 prompt cache |
| 只按固定消息数保留尾部 | 拒绝 | 大工具输出会让门限失真 |

## 待进入设计书的验收问题

1. 压缩前后是否保持最后用户目标、未完成 todo、变更文件、测试状态、错误和下一步？
2. `tool_use/tool_result` 是否永远成组保留？
3. 摘要失败、断连和进程重启后是否仍能恢复原 transcript？
4. 连续压缩是否有节省率、防抖和告警？
5. Prompt cache 是否只在 compact/new session 后失效？
