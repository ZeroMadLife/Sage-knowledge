# V6.6 上下文实现复盘

## 学习目标

把“上下文压缩”拆成可以逐个读源码、跑测试、解释数据流的组件。这里记录的是当前实现，不等同于 V6.7 记忆规划。

## 组件清单

| 组件 | 源码入口 | 当前状态 |
| --- | --- | --- |
| Budget/门限 | `core/coding/context/budget.py` | 已实现，使用 model window 与 output reserve |
| Immutable Projection | `core/coding/context/projection.py` | 已实现，canonical 不被裁剪 |
| Structured Compactor | `core/coding/context/compact.py` | 已实现，摘要 schema、修复、证据校验、冷却/熔断 |
| Controller | `core/coding/context/controller.py` | 已实现，turn-start 压缩与 safe-point 计数 |
| TranscriptStore v2 | `core/coding/persistence/transcript_store.py` | SQLite canonical evidence |
| CompactionStore | `core/coding/persistence/compaction_store.py` | SQLite attempt/checkpoint，HMAC 可选但缺 key 时 fail-closed |

## 最重要的数据流

```text
SQLite canonical transcript
        |
        +--> ContextProjector --> bounded active projection --> ContextManager prompt
        |
        +--> StructuredCompactor --> verified checkpoint --> resume projection
```

**不可混淆的两个历史：**

- canonical transcript：完整、可审计、追加式；不因压缩删除旧 user/tool evidence。
- active projection：给模型看的有限窗口；可以包含 summary，但不是事实来源。

## 生命周期契约

1. turn 边界先读取 canonical/active projection。
2. `context_compaction_started` 先写 CompactionStore，再广播事件。
3. summarizer 只返回严格结构化摘要；失败保留 bounded projection，不能把大原文重新送回模型。
4. completed/failed artifact 持久化后才广播 terminal event。
5. tool loop 只做 projection/recount；emergency 在下一次 `ModelRequested` 前阻断，不在 `finally` 启动压缩。
6. checkpoint 恢复必须通过 HMAC、session/compaction identity 和当前 transcript range/evidence/prefix 校验。

## 已知风险

- event bus 故障后可能留下 stale `started` attempt，需要 V6.7 reconcile。
- legacy/custom history 的额外字段如果不在 TranscriptItem 规范化字段内，重启后 evidence hash 可能回退 canonical。
- `memory_block` 是 prompt 注入层，不能写回 transcript，也不能绕过 projection 预算。
