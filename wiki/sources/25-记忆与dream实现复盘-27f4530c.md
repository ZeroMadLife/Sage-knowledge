# 记忆与 Dream 实现复盘

## 当前源码事实

- `core/coding/memory/working.py`：每次 run 构造 working memory，理论上不跨 run 持久化。
- `core/coding/memory/durable.py`：workspace-scoped Markdown/file durable memory，带基础 source/source_ref。
- `core/coding/memory/manager.py`：组合 working/durable，并持有进程内 pending proposal。
- `core/memory/*`：旧的 Redis/Mem0/travel memory stack，目前没有接入 coding harness，不能与 `core/coding/memory/*` 混为一谈。
- `runtime.py`：每个 turn 将 working+durable memory 作为 `memory_block` 注入 prompt，不写回 canonical transcript。
- `core/coding/persistence/memory_store.py`：已加入 workspace-scoped SQLite proposal/fact/event store，支持 v0 migration、CAS、审批审计和安全路径校验。
- `memory_proposal_ready`：已统一 typed event，并在 proposal 持久化后发送 session/run/reflection/proposal 元数据。

## 当前阻断与明确后续 gap

1. `/remember` 仍有旧 Markdown 路径，尚未完全迁入 SQLite canonical facts。
2. 同内容不同 topic/source 当前按 content hash 去重，会丢多来源 provenance。
3. Markdown projection 与 SQLite 仍是两层状态，后续应统一为 SQLite canonical + 可重建投影。
4. audit event 尚无 hash-chain/外部锚定，不能宣称防篡改审计。
5. Dream proposal 已有 REST list/detail/approve/reject 与前端审核面板，但还没有自动 Dream 调度器和子代理门限。

已关闭：proposal 重启丢失、CAS 审批、重复 approve、stale reject、workspace/link 安全、v0 schema migration、typed proposal event、API/UI 审核闭环。

## V6.7 最小实现顺序

1. SQLite `memory_facts`、`memory_proposals`、`memory_events`，Markdown 只做可重建的人类投影。
2. `remember_explicit(command_id, ...)` 只允许 Runtime command parser 调用；普通 agent tool 只能生成 proposal。
3. Proposal 状态机：`pending -> approved/rejected/expired`，使用 `proposal_id + expected_revision` 原子 CAS。
4. 事件统一为 `session_id/run_id/reflection_id/proposal_id/candidate_count/base_revision`，持久化后再 emit。
5. Context 注入返回稳定排序、token budget、provenance envelope；只注入 active facts。
6. API：proposal list/detail/approve/reject；跨 workspace、旧 revision、重复审批有明确 404/409。
7. 前端显示“待审核”，绝不把 dream proposal 直接当 active memory。

## 验收测试

- 普通 remember tool 不写 durable memory。
- 显式 `/remember` 跨进程重启仍可查。
- proposal 重启后可读，approve/reject 幂等且 CAS 冲突可复现。
- workspace A 的 memory 不会出现在 workspace B。
- stale file/hash、source_ref、status、token budget 都有失败测试。
- proposal event schema 与 contract JSON 一致。
