## 这一版到底做了什么

V7.3.0 解决“普通知识为什么也要逐条审核”：可信本地 Markdown 由确定性解析器处理后自动提交，用户只需要处理外部解析、删除、覆盖冲突和回滚等异常。

V7.3.1 解决“Wiki 有内容但 Agent 搜不到”：每个批准的 page revision 被切成带来源版本的 chunk，写入 SQLite FTS5 与本地 hashing 索引，再用 RRF 合并排名。

V7.3.2 解决最后一公里：网页与 Agent 共同消费同一份 `KnowledgeRetrievalBundle`。用户可以在知识库页面直接提问并查看证据；Coding Runtime 会获得只读 `knowledge_search` 工具，模型可以主动检索这些知识。

## RetrievalBundle 契约

一次检索不会直接返回“模型自由生成的答案”，而是返回受 token budget 限制的证据包：

- `status`：`evidence_found` 或 `no_evidence`；
- `citation_id`：由 workspace、page、page revision、source revision 和 chunk 共同派生；
- `page_revision/source_revision`：回答可以追溯到具体 Wiki 与来源版本；
- `block_id/chunk_id/heading_path`：定位原始解析块和所在章节；
- `sparse_rank/dense_rank/rrf_score`：保留召回与融合顺序；
- `excerpt/token_count/truncated`：按上下文预算提供证据，超限时明确标记截断。

API 与工具都只返回来源相对路径，不返回服务器、Obsidian 或 Git 工作区的绝对路径。

## 无证据门限

`sage.hashing` 是离线、确定性的词法向量 baseline，不具备真正的语义理解。初版验证发现，完全无关的查询也可能因为 hash collision 得到正 cosine，产生伪证据。

当前采取 fail-closed 策略：

1. hashing provider 不允许单独产生 dense-only recall；
2. 命中至少需要通过 FTS 词法证据，dense 只参与重排；
3. 中文多字查询使用 bigram，不再把每个单字都用 `OR` 放大召回；
4. 没有证据时 API 和工具明确返回 `no_evidence`，并要求 Agent 告知用户知识库没有依据。

未来接入真实 embedding provider 后，只有明确声明支持 semantic recall 的 provider 才能开放 dense-only 候选。

## Harness 接线

`CodingRuntime` 新增可选 `knowledge_store`，创建和恢复会话时都从 FastAPI app state 注入。`ToolContext` 将同一实例交给 `knowledge_search`，因此：

- 新会话和 resume 后的旧会话使用相同知识索引；
- 工具是只读、无需审批的 resident tool；
- token budget 默认 3000，top-k 默认 8；
- 知识库未配置时返回 `unavailable`，不会静默改用模型常识；
- 工具结果中的 instruction 明确要求只能基于引用片段声明“来自知识库”。

当前仍是单用户全局 KnowledgeStore。租户隔离、每用户知识空间和公开/私有索引隔离必须在公网多用户前完成。

## UI 验证入口

Knowledge Workspace 新增“向知识库提问”：

- 输入自然语言问题后调用真实检索 API；
- 显示标题、heading path、引用编号、来源相对路径和 page/source revision；
- 显示本次证据 token 用量；
- 无命中时显示“知识库没有找到可用证据”，不伪造答案；
- `390×844` 手机视口下输入区与引用列表无横向溢出。

它目前是“证据检索验证台”，不是最终的 LLM Wiki Chat。下一版会把证据质量门禁、自主沉淀和回滚闭环接入主助手。

## V7.3.3 下一阶段

1. 建立 evidence gate：最低相关性、多来源覆盖、冲突与时效判断；
2. 建立 autonomous learning loop：会话/工具结果形成候选，低风险自动沉淀，高风险仅保留异常审核；
3. 每次沉淀生成新 page revision，支持撤销、回滚、重建索引和审计；
4. 在主助手时间线展示“检索了什么、引用了什么、沉淀了什么”；
5. 增加无答案、冲突事实、旧版本和跨来源 Golden Queries。

本阶段仍不包含 Neo4j/GraphRAG/Louvain、HR 公共窗口、飞书 Bot、Kubernetes 或公网发布。
