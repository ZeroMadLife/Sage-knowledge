# V6.9 交付收口与下一阶段预告

## 本阶段产出边界

V6.9 当前已经完成并进入集成验收的主要内容：

- SQLite 持久时间线、事件序号和 `event_id` 去重；
- 服务端持有的 run、WebSocket 断线补发和历史回放；
- 中文 Coding Workbench：会话管理、深链、工具过程回放、Inspector 与响应式三栏；
- 会话置顶、归档、恢复和重命名等元数据操作；
- 已有 V6.6 上下文压缩、SQLite transcript、Memory Proposal API/UI 的基础能力继续保留。

这不表示 V6.9 已经可以直接合入发布分支。Worktree UI 的最后修复、完整前端测试/构建、响应式浏览器验收及集成分支复审仍是当前收口门禁。

## Git 与 Worktree 团队规则

目标分支结构：

```text
main                 可发布版本，只在 V6.9 全部门禁完成后合入
└── dev/sage-v6      当前唯一集成开发目标
    └── codex/*      短期功能、修复或审查 worktree 分支
```

每一个短期分支完成后，都必须经过以下收口，而不是长期堆积 worktree：

1. 在对应 worktree 跑完与改动匹配的测试、构建和 `git diff --check`。
2. 审查它是否完整包含目标需求、是否有未提交文件、是否误改共享 API/store 边界。
3. 确认分支可安全合并到集成分支，并在集成分支重新跑验证。
4. 确认短期分支已经是集成分支祖先、worktree 干净后，删除该 worktree 和本地短期 `codex/*` 分支。
5. 每次合并后更新本学习库：写明 source commit、测试证据、已关闭风险和未关闭边界。

保留主工作区中用户尚未跟踪的中文设计文档；不能为了清理 git 状态而删除或覆盖用户材料。

## 下一阶段：Skills 与 Agent Harness

下一阶段先做“工具效果与权限执行”，再做路由和安装器，最后接子 Agent。建议顺序：

1. 为工具定义只读/写入效果，并让 `ToolExecutor` 实际校验 `allowed_tools`。
2. 用正式 YAML 解析 Skill manifest；`SAGE.md` 只能用结构化 fenced block 声明已启用 Skill，自由文本不能授予权限。
3. 固定路由优先级：显式 `/skill` > 唯一确定的 auto 命中 > suggest 后用户确认；外部、写操作和歧义 Skill 一律不自动执行。
4. 实现 `sage skill install <local-path|git-url>`，复制资源并检查 manifest、工具映射、依赖、来源和 hash。
5. 添加 `.sage/agents/{explore,reviewer,frontend,backend}.md` profile；V6.9 只启用只读 Explore/Reviewer，frontend/backend 仅可发现、保持禁用。
6. delegate 事件必须带 `parent_run_id`、`agent_run_id`、profile、status，并持久化到时间线；默认禁止子 Agent 继续派生子 Agent。

Dream 不复用通用写权限 Worker，应走专用受限 Reflection Runner。
