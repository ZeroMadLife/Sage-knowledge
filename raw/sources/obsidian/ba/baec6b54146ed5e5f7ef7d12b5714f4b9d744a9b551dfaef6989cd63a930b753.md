# V6.9 中文工作台与会话导航复盘

## 学习目标

掌握 Sage Coding Workbench 怎样把“会话、时间线、过程、检查器”组织成一个可长期使用的中文工作台。交互信息架构参考 Hermes Studio 的密度和工作流，但不复制其 BSL 1.1 源码、样式、资源或品牌。

## 三区布局

```text
桌面端
┌──────────────┬──────────────────────────────┬──────────────────┐
│ 会话栏       │ 对话时间线 + Composer         │ Inspector        │
│ 新建/搜索    │ CodingMessageTurn             │ Files/Changes    │
│ 置顶/归档    │ 工具过程可展开                │ Runs/Memory      │
└──────────────┴──────────────────────────────┴──────────────────┘
```

| 区域 | 已实现内容 | 使用目的 |
| --- | --- | --- |
| 左侧会话栏 | 新建、搜索、重命名、置顶、归档、恢复、运行状态 | 只做工作区和会话导航，不把工具细节堆入侧栏 |
| 中间时间线 | `CodingMessageTurn`、工具过程开关、每会话滚动恢复、长历史分页 | 阅读用户问题、模型回答与可回放执行过程 |
| 右侧 Inspector | Files、Changes、Runs、Memory 四个 tab，可收起、可拖宽 | 在不离开对话的情况下查看工作上下文 |

## 会话元数据和深链

本阶段新增会话元数据接口，session summary 具备 `pinned`、`archived`；默认列表隐藏归档会话并将置顶会话排在前面。前端通过以下方法维护：

- `renameSession`
- `setSessionPinned`
- `setSessionArchived`

路由采用 `/coding/session/:id`，因此浏览器刷新、复制链接或从历史记录返回都可以直接定位指定会话。进入深链时必须先加载会话列表，再调用 `selectSession(id)`；不能先创建一个空白 stub session，否则会污染真实列表并导致用户误以为丢了会话。

## 每会话状态为什么必须分区

页面不是只有一个全局聊天数组。以下内容都按 `session_id` 存储：

- timeline 与 pagination cursor；
- 当前/历史 run、审批、context 与错误；
- 已显示的 `event_id` 集合；
- 滚动锚点和“是否在底部”的阅读位置。

这样用户在会话 A 执行任务时切去 B，再返回 A，能够看到仍在进行的任务和原位置；A 的迟到 WebSocket 事件不会覆盖 B 的界面。

## Inspector 和响应式规则

| 视口 | 左右区域行为 |
| --- | --- |
| 桌面 | 三栏并排；Inspector 可收起、可调宽，宽度和开关写入 localStorage |
| 平板 | Inspector 以抽屉出现，避免压缩中间对话区 |
| 手机 | 左侧会话栏和右侧 Inspector 都是全屏 sheet；中文长文案必须换行，不允许互相遮挡 |

偏好由 `useWorkbenchPreferences.ts` 保存：显示工具过程、Inspector 开关、Inspector 宽度。紧凑屏幕要把“用户偏好中的打开状态”与“当前是否显示为 overlay”分开，避免断点变化后按钮和面板状态不一致。

## 明确未开放的能力

Terminal 在 V6.9 只显示不可用说明，不连接服务器 shell。真正的终端必须等待 V7 的 sandbox、认证、配额和审计边界完成，不能因为 UI 放了 tab 就开放执行入口。

## 建议阅读顺序

1. `frontend/src/views/CodingView.vue`：三栏、断点、路由恢复、滚动和 Inspector 状态。
2. `frontend/src/components/coding/sidebar/CodingSidebar.vue`：会话生命周期操作。
3. `frontend/src/components/coding/inspector/CodingInspector.vue`：四类检查信息与 V7 terminal 占位。
4. `frontend/src/composables/useCodingSessionRoute.ts`：深链读写。
5. `frontend/src/composables/useWorkbenchPreferences.ts`：本地偏好。
6. `frontend/src/stores/coding.ts`：按会话分区和 timeline 事件合并。

## 验收视角

核心场景应始终手工或 E2E 验证：会话 A 执行 read/patch/approval，切到 B 后再回 A 并刷新，工具顺序、参数摘要、结果、状态和最终回答一致且终态只出现一次。视口检查至少覆盖 `1440x900`、`1024x768`、`390x844`，并检查长路径、长命令、键盘焦点、ARIA、抽屉与滚动恢复。
