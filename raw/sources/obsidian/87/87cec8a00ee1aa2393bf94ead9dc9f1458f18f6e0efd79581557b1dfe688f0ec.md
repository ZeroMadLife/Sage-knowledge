## 首页读模型

```text
GET /api/v1/assistant/home
  -> 解析 HttpOnly Sage session（云模式）
  -> CodingSessionStore：owner 可见的近期对话
  -> CloudRepository：owner-scoped project
  -> workspace-scoped MemoryStore：pending proposal 数量
  -> deterministic suggested actions
  -> Cache-Control: no-store
```

`AssistantHomeSummaryService` 不调用模型、不扫描工作区文件、不返回 Memory 正文、Provider Key 或完整路径。生产环境没有有效服务器 session 时返回 401；本地开发模式只显示没有 `owner_user_id` 的本地 session。

知识库在 P1 明确返回 `not_configured`，Wiki 与 Note Proposal 计数为零。这是兼容位，不是假装 RAG 已上线。

## 首条消息链路

```text
AssistantHomeView
  -> codingStore.startSessionWithPrompt(prompt)
  -> POST /coding/session（只创建一次）
  -> 加载 session 真实状态
  -> pendingInitialPrompts[session_id] = prompt
  -> CodingStream.connect()
  -> WebSocket onOpen(session_id)
  -> sendMessage(prompt)
  -> 删除 pending prompt
  -> Router /coding/session/:id
```

prompt 只有在当前 session 的 socket 真正 open 后才发送。发送成功立即从 Map 删除，所以旧 socket 的迟到 open、组件重挂载或 run reconnect 不会重复首条消息。

## 组件职责

| 组件 | 职责 |
| --- | --- |
| `AssistantNavigation.vue` | 左侧能力菜单、移动全屏 Drawer、focus trap 与焦点返回 |
| `AssistantHomeView.vue` | 首页 Composer、summary 加载、创建 session 与路由 |
| `AssistantHomeSummary.vue` | 建议动作、近期对话、项目和 proposal 只读投影 |
| `AssistantSectionView.vue` | Knowledge/Evolution/Public 三个阶段页的共享结构 |
| `assistantHome.ts` | 首页独立状态、请求去重、强制刷新与错误保留 |
| `codingStream.ts` | 增加 session-bound `onOpen`，支撑首条消息一次性发送 |

首页 Store 与 Coding Store 分离。进入 `/assistant` 不创建 session、不连接 WebSocket；只有用户实际发送时才触发 Coding Runtime。

## 视觉与交互

- 左侧 228px 能力菜单，中间最大约 940px 的 Chat-first 内容；
- Sage Green 表示连接和确认，Source Blue 表示来源和导航，Review Yellow 表示待审核，Coral 表示公开/风险边界；
- 保持中性背景，不使用装饰渐变或营销式 Hero；
- `/knowledge`、`/evolution`、`/public` 只展示真实阶段边界，没有假上传、假同步或假发布按钮；
- `1440x900`、`1024x768`、`390x844` 均无横向溢出；
- 手机菜单使用 `aria-modal`，主内容 `inert`，关闭后焦点返回菜单按钮；
- 建议动作 `/assistant?action=compose` 会把焦点送入 Composer。

## 关闭风险与遗留问题

已关闭：

- 首页误创建空 session；
- 首条消息在 WebSocket 未 open 时丢失；
- 重连重复发送首条消息；
- 云 session/project 跨 owner 泄漏；
- 阶段页面伪造 RAG/发布能力；
- 移动菜单遮挡和焦点丢失。

- 当前运行环境没有配置 DeepSeek Key，浏览器未执行真实模型回合；发送一次语义由单元测试覆盖；
- `CodingView` 生产 chunk 仍约 1.08 MB，V7 后续做按功能拆包和虚拟历史列表；
- KnowledgeWorkspace、摄取队列、Wiki diff、RAG 和 HR Agent 均属于 V7-P2 以后；
- cc-connect 由用户独立调试，不进入 P1 至 P3 交付范围。
