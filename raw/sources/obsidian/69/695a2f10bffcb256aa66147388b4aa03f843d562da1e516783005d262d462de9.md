# V6.9 工具轨迹可见性收口

## 本轮交付

- run 折叠摘要优先显示真实动作，例如 `执行 npm test`、`读取 README.md`、`子任务 探索内部工具实现`，不再只显示泛化的工具数量。
- 多次 `tool_search` 合并为一行“工具探索 · N 次”，真实 shell、文件、列表、搜索和子任务步骤优先展示。
- shell 折叠态显示简略命令、状态和步骤耗时；点击“查看输出”后展示命令、退出码、stdout 和 stderr。
- 长命令和文件路径保持单行省略，完整动作通过 `title` 保留。

## 安全边界

- 只展开后端持久化后已脱敏、已限长的 audit preview；活动 run 的原始 timeline 只显示动作摘要，不开放原始结果。
- `read_file`、`write_file`、`patch_file` 不展开文件正文，避免将源码、配置或凭据重新暴露到聊天界面。
- 异常或非预期 JSON preview 会安全降级，不会中断整个 trace 渲染。

## 验证证据

- 前端全量：`27 files / 266 tests passed`。
- 前端生产构建：`npm run build` 通过。
- `git diff --check` 通过。
- 真实 shell 会话：折叠摘要显示 `执行 pwd && printf ...` 与 `41s`；步骤显示 `14ms`、`退出码 0`，展开后可见 stdout/stderr。
- 真实探索会话：5 次 `tool_search` 合并为一行，子任务显示为“探索内部工具实现”；console 无 error/warn。

## 遗留边界

- 异步子 Agent 内部的每一次文件读取和 shell 调用尚未与主 run 建立嵌套 audit 关联，当前只显示子任务名称、状态、耗时和有限结果。
- 生产构建仍有既有的大 chunk 警告，后续由 V7 Web 部署阶段处理分包。
