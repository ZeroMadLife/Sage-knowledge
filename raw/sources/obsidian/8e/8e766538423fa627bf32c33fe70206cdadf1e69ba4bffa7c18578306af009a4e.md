## 学习目标进入 Git truth

`Sage-knowledge/purpose.md` 现在包含结构化 `sage-learning-goal` JSON fenced block。目标不是数据库里的隐式 prompt，而是可读、可审查、可回滚的 Git 文档。

当前总目标：

> 成为具备生产交付能力的全栈 AI 应用工程师。

目标被拆成 7 个能力维度：

1. 前端与产品体验；
2. 后端与数据工程；
3. Agent Harness；
4. RAG 与知识系统；
5. 评测、可观测与安全；
6. 云端交付与运维；
7. 工程闭环与架构决策。

每个维度保存 `capability_id/label/description/keywords/weight/required`。更新接口必须携带 `expected_goal_revision`，并只提交 `purpose.md`，旧 revision 或工作区外部修改会返回冲突。

## 社区分析

V7.5.1 使用 NetworkX 3.5 的 Louvain 实现，并固定：

- `algorithm_id=networkx.louvain`；
- `algorithm_version=3.5`；
- `seed=42`；
- `resolution=1.0`；
- `threshold=1e-7`。

分析 revision 由 `graph_revision + goal_revision + algorithm config` 决定。同一输入重复计算得到同一 analysis revision；只修改学习目标不会重建事实图，只生成新的目标分析。

社区计算排除 Source 叶节点和 `EVIDENCED_BY` 边，只使用 Page/Concept 等知识节点与 `WIKILINK/SHARES_SOURCE`，避免来源节点把聚类拉成噪声星形。

## 关闭风险

- 目标配置拥有 Git revision，不会被 prompt 或数据库静默覆盖。
- 社区算法参数和版本固定，分析可以重放。
- 目标变更与事实图重建解耦。
- Research gap 不自动联网、不自动沉淀，继续服从现有 proposal/revision/citation 链。
- Source 节点不参与 Louvain，降低来源星形结构对社区的干扰。

## 遗留问题

- 关键词覆盖只能说明知识材料出现过相关内容，不能证明工程能力已完成。
- 需要为能力增加代码 commit、测试、部署、文章等验收证据类型。
- 需要区分 implemented、planned、reference，避免计划文档提高“已掌握”分数。
- 前端尚未接 Graph API、社区和 Insight Inspector。
- Web Search、掘金/官方文档连接器和研究预算仍属于 V7.6。
