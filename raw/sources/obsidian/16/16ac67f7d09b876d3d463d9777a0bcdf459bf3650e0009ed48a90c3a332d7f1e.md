# V6.9 工作台运行体验收口

## 收口结论

本阶段已于 2026-07-13 fast-forward 合入 `dev/sage-v6`。Source commit：`bf0971e feat(sage-v6.9): polish chat runtime workbench`。

本轮聚焦 Hermes Studio 对话门面中的运行感受：窗口占满视口、用户消息先于“思考中”、工具在连续时间线中展开、长输出自动跟随但不打断用户上翻，并在右上角显示服务端真实的上下文预算。同时修复了复合验证命令被 `run_shell` 策略误拒绝的问题，没有放宽危险命令审批。

## 完成内容

- 根布局、主工作区、消息区和 Composer 明确占据各自网格行，解决底部大片空白和输入区未贴底。
- 发送后立即显示 optimistic user message，再显示“思考中”、工具和最终回答；服务端时间线确认后无重复消息。
- 流式输出使用单个 `requestAnimationFrame` 合并跟随；手动上翻会停止强制滚动，“回到底部”恢复跟随。
- 工具过程改为连续时间线，每项独立展开/收起，运行中项自动展开，参数和结果支持 JSON/文本高亮、长结果展开与复制反馈。
- 历史分页继续使用 canonical timeline；真实长会话翻到最早页后，`run_shell` 参数、结果和终态完整恢复。
- 新增 `config/coding_models.toml`，模型目录、上下文窗口、输出预留和可用 reasoning mode 由服务端单一来源严格校验。
- DeepSeek Flash/Pro 显式配置 `1,000,000` token 窗口和 `64,000` token 输出预留；`reasoning_modes = []` 时前端不展示伪思考开关。
- 标题栏上下文显示例如 `4.1k / 1.0M · 剩余 995.9k`；手机端收紧为 `4.1k / 1.0M`。
- `SAGE_CODING_MODELS_FILE` 支持部署时覆盖 TOML 路径；文件大小、字段、重复模型、默认模型和 token 边界均 fail-closed。
- 纯 `cat/rg/find/ls` 等普通仓库读取仍强制走专用工具；`python3 --version; pwd; ls -la` 类复合验证命令允许进入原有 shell 审批和危险命令检测链。

## 关闭风险

- 新会话和刷新后工作台不再在底部留出大片空白。
- “准备执行”、“执行过程”和工具详情不再跑到窗口左侧或中间的异常列。
- 用户消息、思考状态、工具与最终回答的时序与实际运行一致。
- 长输出既能自动跟随，也不会在用户阅读历史时强制拉底。
- 工具参数与结果在实时、历史分页和刷新后都有可审计的折叠表达。
- 模型上下文不再依赖前端猜测或环境变量多处拼装，配置失效时服务端拒绝启动。
- 复合 shell 验证命令可用，但危险命令、计划模式写保护、工作区边界和审批链未放宽。

## 遗留问题与下一阶段边界

- 本地文件路径目前只保证不被误识别为公网 URL；点击后打开 Sage 文件预览仍需 workspace file deep-link 契约。云端 Linux 部署不会自动解决浏览器与服务端文件命名空间之间的契约问题。
- 本轮使用 Codex 内置浏览器，未执行 Safari/Firefox/Chrome 全矩阵。
- 长时间线目前每次手动加载 `100` 条事件；本轮证明分页不丢工具，但更友好的按 turn 或自动批量加载属于后续性能与可用性优化。
- 生产包仍有 `CodingView` 大 chunk 提示；Markdown/Highlight.js 按需分包与虚拟长列表留在 V6.9.1。
- 更深的 Hermes 门面对齐应继续聚焦对话核心的字体、间距、窄屏折叠和工具密度，不扩展到与当前主流无关的全量 Studio 功能复制。
- V7 认证和云工作区继续留在独立分支，未混入 V6.9。
