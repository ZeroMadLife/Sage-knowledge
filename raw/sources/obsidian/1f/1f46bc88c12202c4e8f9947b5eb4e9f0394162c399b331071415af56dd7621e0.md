# V6.9 Hermes 门面与工具恢复收口

## 交付结论

本阶段已完成并于 2026-07-13 fast-forward 合入 `dev/sage-v6`。Source commit：`0a57470 feat(coding): finish v6.9 chat workbench polish`。

本轮不是复制 Hermes Studio 源码，而是依据 `http://localhost:8648/#/hermes/chat` 的可观察结构，对齐 Sage 的门面比例、聊天阅读区、输入工作面和 Markdown 节奏。后端同时补齐工具参数恢复与批量预检，解决真实浏览器中新会话无法用鼠标发送、缺少 `path` 时工具链失控等问题。

## 完成内容

- 新浏览器配置默认进入深色模式，并立即持久化用户选择；
- 桌面端去除重复顶栏、缩窄会话栏、扩大聊天正文列；
- Composer 改为一体化底部工作面，发送/停止按钮固定右下并修复透明底栏遮挡点击；
- “准备执行”“执行过程”和工具块统一进入 Sage 正文列，不再跑到窗口左侧或页面中间；
- 时间线投影配对 `model_requested` 与 `model_parsed`，完成运行不再错误显示“进行中”；
- Markdown 标题、列表、引用、行内代码和代码块使用 Sage token；修复全局 CSS 中无效的 `:deep()`；
- 关闭模糊域名识别，`README.md` 不再被渲染成 `http://README.md` 外链；显式 Markdown 链接和带协议 URL 仍可用；
- 工具 schema 在执行、审批和文件访问前预检，缺少必填参数时不产生 `tool_call`，最多纠正两次；
- 批量工具先整体检查 schema 与 workspace path 边界，后置越界参数不会造成前置工具的部分副作用；
- 执行阶段继续二次校验文件状态、权限、策略和审批，没有放宽跨工作区边界。
