## 遗留问题

1. `sentence-transformers` 无约束选择 `torch 2.13.0` 及 CUDA 依赖，导致 API 镜像约 `9.1 GB`；根盘当前约 70% 使用率、剩余约 12 GB。下一版应改为 CPU-only 依赖并在新 release 验证，不能直接修改当前固定 release。
2. `DEEPSEEK_API_KEY` 和外部 OAuth 凭据按受限 staging 边界保持空值，Coding Runtime 当前不可用；凭据必须通过服务器密钥管理补齐，不能写入仓库或日志。
3. 阿里云安全组尚未开放公网 80，外部 TCP 请求超时；这是当前预期安全门，不应在认证、域名和 HTTPS 未完成前开放。
4. Gateway 根路径刻意返回 503：`public access is pending authentication review`。正式私测前需要完成认证入口、域名、HTTPS 与反向代理切换。
5. UFW 仍保留宝塔历史端口（FTP、888、8888、35116、39000:40000 等）；需要确认业务依赖后单独收紧，不能在本次恢复构建中直接删除。
6. Compose 基础服务仍使用可变 tag；后续生产化应固定 digest，并补充部署 manifest、备份恢复演练和 CI/CD。

## 回滚方式

当前没有上一版 `current` 可回切。若 staging 出现问题：

1. 在 `/opt/sage/releases/33e3951` 执行 `docker compose -f compose.yml down` 停止当前栈。
2. 删除 `/opt/sage/current` 符号链接，不删除 release、volume 或 BuildKit 缓存。
3. 需要恢复服务器级配置时，使用 `/var/backups/sage/predeploy-20260715-141557` 中的快照人工比对后恢复，禁止无审查覆盖。
