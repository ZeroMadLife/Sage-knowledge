# V7.0 认证与云工作区控制面复盘

## 本阶段交付了什么

V7.0 只建立云端的**控制面**，没有开始执行用户代码：

```text
浏览器
  -> HttpOnly Cookie
  -> Cloud Login Session (仅保存 SHA-256 hash)
  -> Cloud User
  -> Project (owner_user_id)
  -> Workspace metadata (opaque ID, provisioning)
```

新增 API：

1. `GET /api/v1/cloud/me`：从服务端 cookie session 识别用户。
2. `POST /api/v1/cloud/auth/logout`：撤销 server-side session 并清 Cookie。
3. `POST /api/v1/cloud/auth/dev/login`：只用于本地开发的一次性邀请引导。
4. 项目和 workspace metadata 的创建、读取、列表 API。

## 关键安全边界

- Cookie 原文不入数据库、JSON 响应、日志或模型上下文；数据库只保存 hash。
- development login 必须同时满足 `APP_ENV=development` 和显式开关；production 即使误配开关也返回 404。
- production cookie 强制 `Secure`，并同时使用 `HttpOnly; SameSite=Lax`。
- development identity 完成首次 bootstrap 后不能凭邮箱重复登录，真实重复登录以后由 GitHub OAuth 完成。
- 邀请码通过条件 SQL `UPDATE ... consumed_at IS NULL` 消费；并发请求中只能一个成功。
- Project/Workspace 只能由 owner 读取；猜到其他人的 opaque ID 仍返回 404。
- Workspace 当前只是 metadata：没有目录、路径、clone、runner、shell、terminal 或写工具。
