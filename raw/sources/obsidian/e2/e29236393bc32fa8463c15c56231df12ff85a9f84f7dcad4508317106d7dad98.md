## 关闭风险

- 运行时 usage 数据库不会再被错误标为用户代码改动；
- 回答图标与首个可见 token 同时出现，既不在空消息时抢先显示，也不再被完成动画延迟；
- 页面思考动画只是首 token 前的运行反馈，不代表 provider reasoning；当前 DeepSeek V4 Flash/Pro 配置的 `reasoning_modes = []`，默认不会开启深度推理。
- 工具长输出不再挤占聊天时间线，审计摘要仍保留；
- 每轮变更与其对应回答绑定，不会误打开最近一次其他 run 的 diff；
- 对 provider 的协议兼容不改变首轮工具执行安全边界。
