## 故障现象

知识页显示 `knowledge workspace is not configured`，以下接口持续返回 `503`：

- `GET /api/v1/knowledge/pages`
- `GET /api/v1/knowledge/jobs`

## 根因

根目录 `.env` 虽然启用了 `KNOWLEDGE_JOBS_ENABLED=true`，但没有配置 `KNOWLEDGE_WORKSPACE_ROOT`、`KNOWLEDGE_DATABASE_PATH` 和 `KNOWLEDGE_SOURCE_ROOT`。`create_app()` 因此不会创建 `KnowledgeStore`，接口在 `_require_store()` 处按设计返回 503。

同时机器上还存在两组从已删除 V7 worktree 启动的服务，以及一组切换分支前由 PyCharm 启动的旧 `8000` 后端。旧后端没有当前 V7.2 的知识路由，进一步造成前端与代码版本不一致。

## 本地配置

`.env` 已配置为：

```text
KNOWLEDGE_WORKSPACE_ROOT=/Users/zeromadlife/Desktop/Sage-knowledge
KNOWLEDGE_DATABASE_PATH=/Users/zeromadlife/Desktop/Sage-knowledge/.sage/knowledge.sqlite3
KNOWLEDGE_SOURCE_ROOT=/Users/zeromadlife/Desktop/Obsidian-Knowledge-Base/03_项目/tourswarm/技术沉淀/sage-learning
KNOWLEDGE_SOURCE_ID=sage-learning
KNOWLEDGE_SOURCE_LABEL="Sage Learning"
KNOWLEDGE_SOURCE_KIND=obsidian
KNOWLEDGE_JOBS_ENABLED=true
```

`.env` 被 Git 忽略，本次没有新增、读取或提交任何凭据。

## P2.2-B 开发起点

下一阶段采用小版本直提：

1. P2.2-B1：Parser Registry 与统一 `ParsedDocument` / `ParsedBlock` 契约；
2. P2.2-B2：Markdown、HTML、文本 PDF 的确定性解析；
3. P2.2-B3：MinerU 与 Qwen3-VL 可选适配器、超时与降级；
4. P2.2-B4：Source Understanding、Workspace Synthesis 与 provenance；
5. P2.2-B5：自治 Wiki Policy，低风险自动应用、中风险 draft、高风险审核、敏感内容阻断；
6. 每个小版本在 `dev/sage-v7` 独立测试、提交、推送和复盘。

P2.2-B 不包含 Hybrid RAG、向量索引、GraphRAG 或 HR 公开检索窗口；这些分别属于 P2.3、P2.4 和后续公开知识发布阶段。
