# V7.3.3 证据门禁与自主成长闭环复盘

## 版本证据

- Sage source commit：`e6faefdfa98cb0e7c820eb8b8f4d56c1b1a9c12a`
- 开发分支：`dev/sage-v7`

- 证据生成器：`sage.evidence-learning@1.0.0`
- API：`POST /api/v1/knowledge/learnings`
- Harness 工具：`knowledge_learn`

## 这一版解决了什么

V7.3.2 已经能让网页和 Agent 使用同一个 `knowledge_search` 检索证据，但检索结果仍停留在当前对话，不能转化为以后可复用的知识。

V7.3.3 打通了最小自主成长闭环：

```text
原始文档 -> 版本化切片与索引 -> knowledge_search
         -> 当前 citation 校验 -> 确定性证据快照
         -> autonomy policy -> 自动生成 Git revision
         -> 增量重建索引 -> 后续会话再次检索
```

用户在 Knowledge Workspace 搜索后可以点击一次“沉淀为学习笔记”；Agent 也可以在完成一次有用调查后调用 `knowledge_learn`。两条路径使用完全相同的后端门禁。

## 为什么不再逐条审核

自动沉淀不接受模型写入自由文本事实。调用者只能提交：

- 一个最长 160 字的 topic；
- 1 至 8 个由 `knowledge_search` 返回的 citation ID；
- 可选的 session、run、event provenance。

服务端会重新解析每个 citation，而不是信任浏览器或模型附带的 excerpt。只有同时满足以下条件才自动应用：

1. citation 属于当前 active revision，旧版本引用立即失效；
2. 引用来自 `obsidian`、`markdown`、`github` 或 `feishu` 原始来源；
3. projection 为 private，目标限定在 `wiki/learnings/*.md`；
4. 生成器固定为 `sage.evidence-learning`；
5. 内容只是原始片段的机械整理，不包含模型自由生成的新事实；
6. 每次应用都生成新 page revision、Git commit 和一键撤销元数据。

因此用户无需审核低风险的“证据归档动作”。过期引用、未知引用、重复 citation、越界目标、非私有内容、伪造生成器或派生学习页引用会直接阻断，而不是静默写入。

## 防止递归污染

学习页写回索引后可以被 Agent 检索，用于回答和导航；但是它不能继续作为下一轮自动学习的证据。

这是一个刻意的非对称边界：

- `agent_learning` 内容可以参与 retrieval；
- `agent_learning` citation 不能喂回 `knowledge_learn`；
- 新学习必须重新落到原始 Obsidian、Markdown、GitHub 或飞书材料。

这样可避免同一结论被反复摘录、权重不断放大，最终让系统把自己的旧摘要误认为多份独立证据。

## 持久化与审计

Knowledge SQLite schema 升级到 V7，新增 `knowledge_evidence_learnings`：

- 保存 learning ID、input hash、生成器版本和完整 payload；
- payload 单独保存 SHA-256，读取时验证完整性；
- citation 固化 chunk、page revision、source revision 和 block；
- 同一个 topic 与同一组证据是幂等的，不重复创建 revision；
- `learning_evidence_verified` 事件保存 citation 数量和运行 provenance；
- policy decision 保存 reason codes、applied revision 和 undo 信息。

同一 topic 后续使用一组不同的当前证据时，会在稳定 Wiki page 上追加新的 evidence snapshot，形成可审计的成长轨迹。

## UI 与 Harness

Knowledge Workspace 的检索结果底部新增“沉淀为学习笔记”：

- 点击后直接完成证据校验、策略判断、Git projection 和索引更新；
- 成功时显示可撤销学习页；
- 引用失效或来源不合规时显示真实后端错误；
- 手机视口下按钮改为整行，不产生横向溢出。

Coding Runtime 新增无需人工审批的 `knowledge_learn` resident tool。它之所以可以免审批，不是因为允许 Agent 任意写入，而是工具 schema 不存在 fact/content 字段，后端只能从可信 citation 重建内容。

## 已关闭风险

- 模型自由文本不能直接变成 canonical knowledge；
- 过期 citation 不能在来源更新后继续沉淀；
- 学习页不能自我引用形成反馈放大；
- 自动动作都有 Git revision、policy decision、event 和 undo；
- API 与工具不返回 Obsidian 或服务器的绝对路径；
- 未配置知识库时工具 fail closed，不回退为“假装检索成功”。

## 遗留问题与下一阶段边界

当前 learning 是“可信证据快照”，还不是真正的多文档知识综合。下一阶段需要：

1. 增加多来源覆盖、冲突事实和时效判断，避免只凭一条片段下结论；
2. 让 LLM 只在结构化 claim schema 中综合，并为每条 claim 绑定 citation；
3. 增加 citation coverage checker，任何无引用句子都不能自动进入 canonical Wiki；
4. 在助手时间线展示检索、引用、沉淀和撤销事件；
5. 扩充无答案、冲突、旧版本和跨文档 Golden Queries；
6. 公网前完成 workspace/tenant 隔离与 PostgreSQL + pgvector 迁移。

Neo4j/GraphRAG/Louvain、HR 公共门面、飞书 Bot、Kubernetes 和公网生产部署仍不属于 V7.3.3。
