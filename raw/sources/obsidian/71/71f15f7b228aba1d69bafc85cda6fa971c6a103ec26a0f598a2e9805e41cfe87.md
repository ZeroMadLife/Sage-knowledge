## 1. 本阶段解决了什么

V7-P1 的个人助手首页已经能展示真实项目、会话和沉淀状态，但首页和 Coding Chat 分别创建自己的页面壳。从首页发送第一条消息后，用户只能看到一次突兀的路由切换。同时，首页、导航和工具过程大量使用 `9px` 到 `12px` 字号，中文阅读负担较大。

V7-P1.1 完成三项收束：

1. assistant、coding、knowledge、evolution、public 路由共享应用级 `AssistantNavigation`；
2. 首页 Composer 通过原生 View Transition 形变为 Chat 底部 Composer；
3. 主要工作台字号收敛为语义 Token，正文提高到 `15px`，元信息最低 `12px`。

## 2. 组件职责

| 组件/模块 | 当前职责 |
| --- | --- |
| `App.vue` | 根据 route meta 选择共享 Sage Shell 或独立设置布局 |
| `router/index.ts` | 为个人助手相关路由声明 `assistantShell` 元数据 |
| `AssistantNavigation.vue` | 持久主菜单、移动端全屏菜单与焦点圈定 |
| `AssistantHomeView.vue` | 创建带首条消息的新 session，并触发 Composer 形变 |
| `useViewTransition.ts` | 封装浏览器能力检测、reduced-motion 降级与 DOM 更新等待 |
| `CodingView.vue` | Chat 时间线、标题栏和按需会话 Drawer，不再常驻第二侧栏 |
| `CodingComposer.vue` | Chat 输入、Provider/权限/上下文控制及 shared-element 终点 |

## 3. 关键事件序列

```text
用户提交首页 Prompt
  -> startSessionWithPrompt
  -> 后端返回 session_id
  -> pendingInitialPrompts 保存首条消息
  -> runViewTransition(router.push)
  -> 首页 Composer 形变到 Chat Composer
  -> CodingView 恢复 timeline 并连接 WebSocket
  -> WebSocket open 后首条消息只发送一次
  -> optimistic user message / tool trace / final 进入时间线
```

View Transition 只是视觉增强，不拥有业务状态。session 创建失败时不会触发动画，输入草稿仍留在首页；浏览器不支持 API 或用户开启减少动态效果时直接导航。

## 4. 移动端交互修正

首次浏览器检查发现全局菜单按钮和会话 Drawer 按钮同时占据左上角。最终约定：

- 左上角汉堡按钮只负责全局能力菜单；
- Chat 标题栏右侧 History 图标负责会话列表；
- 两个 Drawer 都支持 Escape、焦点圈定和焦点返回；
- 手机会话 Drawer 占满 Chat 区域，标题和 Composer 不被遮挡。

## 6. 关闭风险

- 首页和 Chat 不再出现两套常驻左栏；
- 首条消息的单次发送语义未被动画接管；
- session 创建失败不会清空草稿；
- 移动端两个入口不再重叠；
- 主要 Chat 可见文字不再使用 `9px`、`10px` 或 `11px`。

## 7. 遗留问题

- View Transition 是渐进增强，非 Chromium 浏览器主要依靠无动画降级；
- CodingView 生产 chunk 仍超过 500kB，属于既有拆包问题，放入后续性能切片；
- KnowledgeWorkspace 和 Wiki 页面仍是 V7-P2 边界，当前首页只显示真实的未配置状态；
- 云端浏览器不能直接读取本机 Obsidian Vault，任意本地目录接入保留到 V8 Local Companion。
